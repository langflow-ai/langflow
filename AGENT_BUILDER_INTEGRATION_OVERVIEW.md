### **Q1: What happens when a user query comes in?**
**SEQUENTIAL EXECUTION:**

3. **LLM Analysis** ‚Üí GPT-4 analyzes prompt for healthcare requirements
4. **Task Decomposition** ‚Üí Breaks down into subtasks with capabilities needed
5. **Component Search** ‚Üí Azure Search finds matching components by capabilities
6. **Assembly & Validation** ‚Üí Healthcare compliance checking
7. **YAML Generation** ‚Üí Creates deployable Langflow configuration
8. **Streaming Events** ‚Üí Real-time updates sent back via SSE

### **Q4: How is final YAML handled?**

**CURRENT IMPLEMENTATION:**
- **YAML GENERATED** during Event 11 ("Generating production-ready agent YAML...")
- **YAML SENT** in Event 12 (`complete`) within `data.workflow.yaml_config`
- **Frontend RECEIVES** YAML as string in streaming response
- **Frontend DECIDES** what to do: display, save, deploy, download

**NOT AUTOMATICALLY STORED** - Frontend handles persistence.
