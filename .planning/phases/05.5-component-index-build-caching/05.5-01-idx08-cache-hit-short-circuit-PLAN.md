<objective>
Short-circuit `ComponentCache.get_and_cache_all_types_dict` on the cache-hit path so a cold-start with a fresh, version-matched user cache returns the cached `all_types_dict` without walking `lfx.components` or calling `import_langflow_components`. Reuse the cache blob already loaded by the IDX-07 peek (no second file read). On cache-miss (file absent, version mismatch, corrupt, or missing `entries`), fall through to the existing rebuild path unchanged. This satisfies ROADMAP Phase 5.5 success criteria 1 and 3 (IDX-08) and honors CONTEXT.md D-01, D-02, D-03, D-06, D-07.
</objective>

<context>

# Binding decisions (CONTEXT.md)

- **D-01** Cache-hit = `cache_path.exists()` AND `installed_version is not None` AND blob parses as dict with non-empty `entries` AND `cached_version == installed_version`. Any miss -> fall through silently.
- **D-02** Short-circuit sits inside `get_and_cache_all_types_dict`, AFTER the IDX-07 peek, INSIDE `async with component_cache.lock:` under a double-check (`if component_cache.all_types_dict is None`).
- **D-03** The short-circuit must not regress `TestIDX01LazyLock::test_cache_built_once_threading` (concurrency invariant).
- **D-06** Silent fall-through. No new log on miss. `_read_component_index` and the IDX-07 warning remain the source of truth for cache-state logging.
- **D-07** Skip telemetry on the cache-hit path (no build work happened; a "0ms build" metric is noise). `import_langflow_components` is the only caller of `_send_telemetry` in this flow, so NOT calling it is sufficient -- no explicit skip logic needed.

# Research pitfalls (RESEARCH.md)

- **P-1** Shape mismatch. Cache blob has `{"version", "metadata", "entries": [[top_level, components], ...], "sha256"}`. `component_cache.all_types_dict` expects flat `{top_level: components, ...}`. Reconstruct via the same `entries` iteration pattern at `components.py:348-354`, then apply `filter_disabled_components_from_dict` (already imported at `components.py:19`).
- **P-2** `_build_code_hash_lookups(component_cache)` MUST run on the cache-hit path. Without it, `component_cache.type_to_current_hash` stays `None` and `ensure_component_hash_lookups_loaded` triggers a spurious second rebuild.
- **P-4** The IDX-07 peek's `cached_blob` local is only assigned on clean parse. Structure the code so the short-circuit activates ONLY when the four D-01 conditions pass.

                _build_code_hash_lookups(component_cache)
