<Icon name="Blocks" aria-hidden="true" /> [**Bundles**](/components-bundle-components) are groups of components that are related to a specific service provider.
If you want to contribute your custom components back to the Langflow project, you must put them into a bundle.

Follow these steps to add components to <Icon name="Blocks" aria-hidden="true" /> **Bundles** in the Langflow visual editor.
This example adds a bundle named `DarthVader`.

## Where to put your bundle code (backend)

_Note:_ the repository layout changed — components are now located at `src/lfx/src/lfx/components/` (previously `src/backend/base/langflow/components/`). Place your new bundle folder under the new location so the loader can discover it.

1. Create a new folder for your bundle at:
   `src/lfx/src/lfx/components/darth_vader`

2. Within `darth_vader`, add the backend files for your component(s). Example structure:

   - src/lfx/src/lfx/components/darth_vader/
     - __init__.py
     - components.py
     - darth_vader_component.py (optional separate files per component)

### Minimal backend component example

src/lfx/src/lfx/components/darth_vader/components.py

```python
from typing import Any, Dict
# Import the appropriate base component class; adjust the import path if necessary
from lfx.components.base import BaseComponent

class MyHelloComponent(BaseComponent):
    """A tiny example component."""

    type = "my_hello_component"
    name = "Hello (MyBundle)"

    def run(self, inputs: Dict[str, Any]) -> Dict[str, Any]:
        message = inputs.get("message", "Hello")
        return {"output": f"{message}, from MyHelloComponent"}
```

### Expose components for discovery

In src/lfx/src/lfx/components/darth_vader/__init__.py expose a discovery function:

```python
from .components import MyHelloComponent

def get_components():
    """
    Return a list of component classes to register.
    The loader will import the module or call this function to discover components.
    """
    return [MyHelloComponent]
```

If the project uses a registry/manifest, ensure it references the module path (for example `lfx.components.darth_vader`) according to packaging.

## Frontend: adding an icon for the bundle

If you want an icon shown in the editor sidebar, add the icon under the frontend icons location:

Recommended path:
- src/frontend/src/icons/DarthVader/
  - darth_vader-icon.svg (raw SVG)
  - DarthVaderIcon.jsx (JSX component)
  - index.tsx (wrapper/TSX export)

Example JS / JSX and TSX patterns (adjust to existing project conventions):

DarthVaderIcon.jsx
```javascript
const DarthVaderIcon = (props) => (
  <svg xmlns="http://www.w3.org/2000/svg" width={24} height={24} viewBox="0 0 32 32" fill="none" style={{ backgroundColor: "#9100ff", borderRadius: "6px" }} {...props}>
    <g transform="translate(7, 7)">
      <path /* ... */ fill={props.isdark === "true" ? "white" : "black"} />
    </g>
  </svg>
);

export default DarthVaderIcon;
```

index.tsx
```typescript
import { useDarkStore } from "@/stores/darkStore";
import React, { forwardRef } from "react";
import DarthVaderIconSVG from "./DarthVaderIcon";

export const DarthVaderIcon = forwardRef<SVGSVGElement, React.PropsWithChildren<{}>>((props, ref) => {
  const isdark = useDarkStore((state) => state.dark).toString();

  return <DarthVaderIconSVG ref={ref} isdark={isdark} {...props} />;
});

export default DarthVaderIcon;
```

To lazily import the icon, add an entry to the icons loader file (example path):
- src/frontend/src/icons/lazyIconImports.ts

Example addition:
```typescript
  DarthVader: () =>
    import("@/icons/DarthVader").then((mod) => ({ default: mod.DarthVaderIcon })),
```

## Register the bundle in the Bundles sidebar

Update the bundles list used by the frontend to show your new bundle. The older docs referenced a sidebar entry that pointed to `/src/backend/...` — update that to reference the new backend bundle folder name.

Example (where the project defines SIDEBAR_BUNDLES or similar):
```typescript
{ display_name: "DarthVader", name: "darth_vader", icon: "DarthVader" },
```

- `display_name` — label shown in the UI
- `name` — folder name under `src/lfx/src/lfx/components/` (darth_vader)
- `icon` — the icon key used by the frontend

## Update bundle component icon reference

In your backend component class, point the `icon` attribute to the icon name you created:

```python
class DarthVaderAPIComponent(LCToolComponent):
    display_name: str = "Darth Vader Tools"
    description: str = "Use the force to run actions with your agent"
    name = "DarthVaderAPI"
    icon = "DarthVader"
```

## Build & test locally

1. Rebuild backend and frontend as required by the project (example commands from repo):
   - make install_frontend && make build_frontend && make install_backend && uv run langflow run --port 7860
   (Adjust to current repository build steps if different.)

2. Reload the frontend and verify the new bundle appears under Bundles.

3. If components do not appear, check server logs for import errors and confirm:
   - The module path and package is importable (consider pip install -e . in repo root)
   - `get_components()` is present and returns the component list
   - The folder name matches the `name` used in the sidebar registration

## Troubleshooting

- Search the codebase for any hardcoded references to `src/backend/base/langflow/components/` and update them if they are still present.
- If a different registration/loader pattern is required by your version of the repo, inspect the component loader code (search for `get_components`, `components`, `SIDEBAR_BUNDLES`, or the code that builds the components registry) and adjust examples accordingly.
