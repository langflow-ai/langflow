[Bundles](/components-bundle-components) contain custom components that support specific third-party integrations with Langflow.

This page describes the components that are available in the **Mem0** bundle.

## Mem0 OSS Chat Memory

The **Mem0 OSS Chat Memory** component retrieves and stores chat messages using Mem0 OSS (open source) memory storage. This is a self-hosted solution that provides full control over configuration and data storage.

## Mem0 Platform Chat Memory

The **Mem0 Platform Chat Memory** component retrieves and stores chat messages using Mem0 Platform (hosted) memory storage. This is a managed solution with enterprise features and automatic updates.

### Mem0 OSS Chat Memory parameters

### Mem0 Chat Memory outputs

Both **Mem0 OSS Chat Memory** and **Mem0 Platform Chat Memory** components can output either **Mem0 Memory** ([`Memory`](/data-types#memory)) or **Search Results** ([`Data`](/data-types#data)).
You can select the output type near the component's output port.

Use the **Mem0 Memory** output for memory storage and retrieval operations with the [**Message History** component](/components-helpers#message-history).

Use the **Search Results** output to retrieve specific memories based on a search query.

### Choosing between OSS and Platform

- **Mem0 OSS Chat Memory**: Choose this for self-hosted deployments where you need full control over data storage, configuration, and infrastructure. Requires setting up vector stores, databases, and LLM configurations.

- **Mem0 Platform Chat Memory**: Choose this for a managed solution with enterprise features, automatic updates, and SOC 2 & HIPAA compliance. Only requires an API key to get started.
