[Agentics](https://github.com/IBM/agentics/) uses LLMs to transform tabular data: add or fill columns row-by-row (map), collapse many rows into one (reduce), filter rows by natural-language conditions, or generate synthetic rows. For example, turn a table of customer reviews into sentiment scores per row, or into a single summary.

The **Agentics** bundle in Langflow provides components that run these operations inside your flows.

## Prerequisites

1. Install the Agentics package in Langflow's virtual environment.
```bash
uv pip install agentics-py
```

2. Restart Langflow so the Agentics components are available:

```bash
uv run langflow run
```

### SemanticMap

Processes a dataframe and expands it by adding new columns generated by LLMs (one row in, one row out per row).

#### Semantic Map parameters

| Name | Type | Description |
|------|------|-------------|
| Language Model | String | Input parameter. Specifies the name of the LLM to be used. Use guided experience. |
| Source DataFrame | DataFrame | Input parameter. Input dataframe (JSON list of dicts or DataFrame). |
| Generated Fields | Table | Input parameter. Describe the output columns to be generated. |
| Instructions | String | Input parameter. Instructions for generating the new column values. |
| Append To Source Columns | bool | Input parameter (advanced). If false, returns only the new columns; if true, appends them to the original data. Default: true. |

**Generated Fields** is defined by editing a table in which each row specifies:
- **name** — the name of the column to be generated
- **type** — one of: str, int, float, bool, dict
- **description** — optional instructions for the LLM to generate the column's content
- **As List** — boolean; if true, the column is a list of values (e.g. entities extracted from text)

### Semantic Aggregator

Aggregates all rows in the input dataframe into a single row following the desired output schema.

#### Semantic Aggregator parameters

| Name | Type | Description |
|------|------|-------------|
| Language Model | String | Input parameter. Specifies the name of the LLM to be used. |
| Source DataFrame | DataFrame | Input parameter. Input dataframe (JSON list of dicts or DataFrame). Required. |
| Generated Fields | Table | Input parameter. Describe the output columns to be generated. |
| Instructions | String | Input parameter (advanced). Optional instructions for generating the aggregated values. |

**Generated Fields** is defined by editing a table in which each row specifies:
- **name** — the name of the column to be generated
- **type** — one of: str, int, float, bool, dict
- **description** — optional instructions for the LLM
- **As List** — boolean; if true, the column is a list of values

### Semantic Filter

Filters out rows from a dataframe based on LLM-evaluated conditions.

#### Semantic Filter parameters

| Name | Type | Description |
|------|------|-------------|
| Language Model | String | Input parameter. Specifies the name of the LLM to be used. Use guided experience. |
| Source DataFrame | DataFrame | Input parameter. Input dataframe (JSON list of dicts or DataFrame). |
| Predicate Template | String | Input parameter. Expression or natural-language predicate the LLM evaluates per row. If true, the row is kept; otherwise it is filtered out. |
| Batch Size | Integer | Input parameter (advanced). Number of rows to evaluate per batch. Default: 10. |

Predicate template examples:
- `is about war` — the whole input row is evaluated if no explicit `{field_name}` is used (can lead to high token usage)
- `{temperature} is higher than 30` — only the `temperature` column is used; other columns are discarded
- `{content} is about war and {country} is in Europe` — multiple columns can be used in the predicate

### Synthetic Data Generator

Generates fake data of the requested type using LLMs.

#### Synthetic Data Generator parameters

| Name | Type | Description |
|------|------|-------------|
| Language Model | String | Input parameter. Specifies the name of the LLM to be used. Use guided experience. |
| Generated Fields | Table | Input parameter. Describe the output columns to be generated (schema). |
| Number of Instances | Integer | Input parameter. How many synthetic rows to generate. Default: 10. |

### Agentics

General-purpose component that can perform Map/Reduce operations over dataframes based on user-defined instructions. For details on Agentics and logical transduction, see the [Agentics documentation](https://ibm.github.io/Agentics/).

#### Agentics parameters

| Name | Type | Description |
|------|------|-------------|
| Language Model | String | Input parameter. Specifies the name of the LLM to be used. Use guided experience. |
| Source DataFrame | DataFrame | Input parameter. Input dataframe (JSON list of dicts or DataFrame). |
| Transduction Type | Dropdown | Input parameter. Options: amap, areduce, generate. Required. |
| Generated Type | String | Input parameter. Name for the generated target type. Required. |
| Generated Fields | Table | Input parameter. Describe the output columns to be generated. |
| Instructions | String | Input parameter. Instructions on how to perform the transduction. |
| Merge Source States | bool | Input parameter (advanced). If true, source row fields are appended to the output; if false, only the target type is returned. Default: true. |
| Batch Size | Integer | Input parameter (advanced). For amap: rows per batch; for generate: number of instances. Default: 10. |

- **amap** — all rows are processed concurrently; output has the same number of rows as the source.
- **areduce** — all rows are processed together; output is a single row of the specified type.
- **generate** — no source dataframe is required; multiple fake instances of the specified type are generated.
