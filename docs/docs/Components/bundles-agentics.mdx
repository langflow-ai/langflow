[Agentics](https://github.com/IBM/agentics/) is a library that enables data transformation over dataframe using LLMs in a map/reduce style computation paradigm.

You can use Agentics to expand a dataframe by adding new columns generated by LLMs, to aggregate multiple rows into a single output row based on LLM-evaluated conditions, or to generate synthetic data of a specified type.

For example, you can process a dataframe of customer reviews and generate sentiment scores for each review or aggregate product descriptions into a single summary.

## Agentics Bundle

The Agentics bundle in Langflow provides the following components:
- **Semantic Map** each row of the input dataframe is expanded by adding new columns adhering to the defined schema
- **Semantic Aggregation** multiple rows of the input dataframe are considered together to generate a single output row adhering to the defined schema
- **Syntetic Data Generation** generate fake data adhering to the defined schema
- **Semantic Filter** filter rows in a dataframe based on LLM-evaluated conditions expressed in natural language
- **Agentics** generalized component that can perform map/reduce operations over dataframes based on user defined transduction instructions, subsuming the previous components

### SemanticMap

This component allows you to process a large dataframe and expand it by adding new columns generated by LLMs.
It works as both mappper and aggregator, enabling you to generate new columns for each row or aggregate multiple rows into a single output row.

#### Semantic Map parameters


| Name | Type | Description |
|------|------|-------------|
| Language Model | String | Input parameter. Specifies the name of the LLM to be used. Use guided experience |
| Source Dataframe | DataFrame | Input parameter. Input dataframe|
| Generated Fields | Table | Input parameter. Describe the output columms to be generated |
| Instruction | String | Input parameter. Specific instruction on how to generate new columns from source |
| Aggregate Multiple Rows| bool | Input parameter. If false, it process each rown concurrently and generates new rows, if true it process all rows at once and return a single row|



**Generated Fields** parameter is defined by editing a table in which each row specifies the following
- name (the name of the column to be generated), 
- type (selected from list of supported types, including String, Integer, Float, Boolean),
- description (optional, it is a set of instruction for the LLM to generate the column's content),
- as_list, a boolean indicating whether the column is a list of values or a single value (can be used to generate multiple values, e.g. entites extracted from a text),  


### Semantic Filter

This component allows you filter out rows from a dataframe based on LLM-evaluated conditions.

#### Semantic Filter parameters



| Name | Type | Description |
|------|------|-------------|
| Language Model | String | Input parameter. Specifies the name of the LLM to be used. Use guided experience |
| Source Dataframe | DataFrame | Input parameter. Input dataframe|
| Predicate Template | String | Input parameter. An expression comprising values from the processed row that is evaluated for truthness . If positive , the rows is selected otherwise filtered out|

Predicate template examples: 
- is about war - the whole input row is evaluated against the predicate if no explicit \{field_name\} is used (be careful with this as it may lead to high token usage)
- \{temperature\} is higher than 30 - where temperature is a column in the dataframe , only the selected column is used in the predicate, all others are discarded
- \{content\} is about war and \{country\} is in Europe  - where content and country are columns in the dataframe, which is a series of articles, multiple columns can be used in the predicate


### Synthetic Data Generator

This component allows you filter to generate fake data of the requested type using LLMs.

#### Synthetic Data Generator parameters

| Name | Type | Description |
|------|------|-------------|
| Language Model | String | Input parameter. Specifies the name of the LLM to be used. Use guided experience |
| Generated Fields | Table | Input parameter. Describe the output columms to be generated |
| Number of generated instances | Integer | Input parameter. The number of instances that will be generated |


### Agentics

This component generalizes the previous components by allowing you to define a custom Agentics objects that can perform map/reduce operations over dataframes.
For more information about Agentics and logical transduction, see the [Agentics documentation](https://ibm.github.io/Agentics/).
#### Agentics parameters

| Name | Type | Description |
|------|------|-------------|
| Language Model | String | Input parameter. Specifies the name of the LLM to be used. Use guided experience |
| Source Dataframe | DataFrame | Input parameter. Input dataframe|
| Transduction Type | Options List | Input parameter. options are amap, areduce and generate |
| Generated Type | String | Input parameter. The name of the output type |
| Generated Fields | Table | Input parameter. Describe the output columms to be generated |
| Instructions | String | Input parameter. Specific instructions on how to perform the transduction |
| Prompt Template | String | Input parameter. A langchain style prompt template applied to the source data to provide input for transduction |
| Merge Source States| bool | Input parameter. If true, it appends the source row fields to the generated output row, if false it return a dataframe of the target type |

- When uses in **amap** mode, all rows are processed concurrently, returning a dataframe with the same number of rows as the source dataframe.
- When used in **areduce** mode, all rows are processed together, returning a single output row of the specified type.
- When used in **generate** mode, no source dataframe is required, and it generates multiple fake instances of the specified type.


## Prerequisites
To use the Agentics bundle in Langflow, you need to install the Agentics package in your Langflow environment. You can do this by running the following command:

```bash
cd langflow
uv pip install agentics-py
```
now restart langflow to use the Agentics components:

```bash
uv run langflow run
```

## Learn more about Agentics
[Agentics](https://github.com/IBM/Agentics) is a Python framework for structured, scalable, and semantically grounded agentic computation. 
It allows you define AI-powered pipelines as typed data transformations—combining Pydantic schemas, LLM-powered transduction, and async execution. 

To learn more about Agentics:
- visit the [official documentation](https://ibm.github.io/Agentics/)
- read this paper 
[Transduction is All You Need for Structured Data Workflows](https://arxiv.org/abs/2508.15610)
- Agentics source code is available on [github](https://github.com/IBM/agentics/)
