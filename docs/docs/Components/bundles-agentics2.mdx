[Agentics](https://github.com/IBM/agentics/) is a library that enables data transformation over dataframe using LLMs in Map/Reduce style programming model.

Think of an LLM as a black box that generates any text that you request, and you can generate dataframes or structured data.
You can load dataframes into Agentics and then use LLMs to do many interesting things, map input dataframe into another dataframe of desired structure, aggregate the context into structured output following the desired schema, or generate synthetic dataframes of desired structure.
The basic mental model is that you play with structured data (internally Pydantic models) and LLMs are like a function with structured input and output types (in Agentics, it is called transducible functions).


For example, you can process a dataframe of customer reviews and generate sentiment scores for each review or aggregate the review statistics or summarize product descriptions.


## Agentics Bundle

Agentics bundle in Langflow provides the following components:
- **Semantic Map** given input dataframe, LLM adds new columns of desired types for each row. For example LLM can generate answer column to all questions in the dataframe
- **Semantic Aggregation** multiple rows of the input dataframe are aggregated and reduced into a single output row adhering to the defined schema
- **Semantic Filter** filter rows in a dataframe based on the predicates in natural language that LLM evaluates
- **Synthetic Data Generation** generate synthetic data adhering to the defined schema
- **Agentics** generalized component that can perform Map/Reduce operations over dataframes based on user defined instructions, subsuming the previous components

### SemanticMap

This component allows you to process a large dataframe and expand it by adding new columns generated by LLMs.


#### SemanticMap parameters


| Name | Type | Description |
|------|------|-------------|
| Language Model | String | Input parameter. Specifies the name of the LLM to be used. |
| Source Dataframe | DataFrame | Input parameter. Input dataframe|
| Generated Fields | Table | Input parameter. Describe the output columns to be generated |
| Instruction | String | Input parameter. Optional instruction on how to generate new columns from source |


**Generated Fields** parameter is defined by editing a table in which each row specifies the following
- Name (the name of the column to be generated)
- Description (optional, it is a set of instructions for the LLM to generate the column's content)
- Type (selected from list of supported types, including String, Integer, Float, Boolean)
- As List, a boolean indicating whether the column is a list of values or a single value (can be used to generate multiple values, e.g. entities extracted from a text)

### SemanticFilter

This component allows you to filter out rows from a dataframe based on expressions or natural language predicates evaluated by LLM.


#### SemanticFilter parameters



| Name | Type | Description |
|------|------|-------------|
| Language Model | String | Input parameter. Specifies the name of the LLM to be used. Use guided experience |
| Source Dataframe | DataFrame | Input parameter. Input dataframe|
| Predicate Template | String | Input parameter. An expression or a natural language predicate that LLM can evaluate to determine whether each row is true or false. If true, the rows is selected; otherwise filtered out|

Predicate template examples: 
- is about war - the whole input row is evaluated against the predicate if no explicit \{field_name\} is used (be careful with this as it may lead to high token usage)
- \{temperature\} is higher than 30 - where temperature is a column in the dataframe , only the selected column is used in the predicate, all others are discarded
- \{content\} is about war and \{country\} is in Europe  - where content and country are columns in the dataframe, which is a series of articles, multiple columns can be used in the predicate

### Agentics

This component generalizes the previous components by allowing you to define a custom Agentics object that can perform Map/Reduce operations over dataframes.
For more information about Agentics and logical transduction, see the [Agentics documentation](https://ibm.github.io/Agentics/).

#### Agentics parameters

| Name | Type | Description |
|------|------|-------------|
| Language Model | String | Input parameter. Specifies the name of the LLM to be used. Use guided experience |
| Source Dataframe | DataFrame | Input parameter. Input dataframe|
| Transduction Type | Options List | Input parameter. options are amap, areduce and generate |
| Generated Type | String | Input parameter. The name of the output type |
| Generated Fields | Table | Input parameter. Describe the output columns to be generated |
| Instructions | String | Input parameter. Specific instructions on how to perform the transduction |
| Prompt Template | String | Input parameter. A langchain style prompt template applied to the source data to provide input for transduction |
| Merge Source States| bool | Input parameter. If true, it appends the source row fields to the generated output row, if false it returns a dataframe of the target type |

- When used in **amap** mode, all rows are processed concurrently, returning a dataframe with the same number of rows as the source dataframe.
- When used in **areduce** mode, all rows are processed together, returning a single output row of the specified type.
- When used in **generate** mode, no source dataframe is required, and it generates multiple fake instances of the specified type.


## Prerequisites
To use the Agentics bundle in Langflow, you need to install the Agentics package in your Langflow environment. You can do this by running the following command:

```bash
cd langflow
uv pip install agentics-py
```
now restart langflow to use the Agentics components:

```bash
uv run langflow run
```

## Learn more about Agentics
[Agentics](https://github.com/IBM/Agentics) is a Python framework for structured, scalable, and semantically grounded agentic computation. 
It allows you to define AI-powered pipelines as typed data transformations—combining Pydantic schemas, LLM-powered transduction, and async execution. 

To learn more about Agentics:
- visit the [official documentation](https://ibm.github.io/Agentics/)
- read this paper 
[Transduction is All You Need for Structured Data Workflows](https://arxiv.org/abs/2508.15610)
- Agentics source code is available on [github](https://github.com/IBM/agentics/)
