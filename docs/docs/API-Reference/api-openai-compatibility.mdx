Langflow provides OpenAI-compatible API endpoints that allow you to use your Langflow flows as if they were OpenAI models. This enables seamless integration with existing OpenAI-compatible applications and libraries.

## Endpoint

```
POST /api/v1/responses
```

## Authentication

All requests require an API key passed in the `x-api-key` header:

```bash
curl -X POST \
  "$LANGFLOW_SERVER_URL/api/v1/responses" \
  -H "x-api-key: $LANGFLOW_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "your-flow-id",
    "input": "Hello, how are you?"
  }'
```

## Request

### Request Body

```json
{
  "model": "your-flow-id",
  "input": "Your input text here",
  "stream": false,
  "previous_response_id": null,
  "include": null
}
```

### Request Headers

| Header | Required | Description |
|--------|----------|-------------|
| `x-api-key` | ✅ | Your Langflow API key |
| `Content-Type` | ✅ | Must be `application/json` |
| `X-LANGFLOW-GLOBAL-VAR-*` | ❌ | Global variables (optional) |

## Parameters

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `model` | string | ✅ | - | The flow ID to execute (used instead of OpenAI model name) |
| `input` | string | ✅ | - | The input text to process |
| `stream` | boolean | ❌ | `false` | Whether to stream the response |
| `background` | boolean | ❌ | `false` | Whether to process in background |
| `tools` | array | ❌ | `null` | **Not supported yet** - will return error if provided |
| `previous_response_id` | string | ❌ | `null` | ID of previous response to continue conversation |
| `include` | array[string] | ❌ | `null` | Additional response data to include (e.g., `['tool_call.results']`) |
